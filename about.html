About Our Page

Our mission is to construct a website that offers several products with style and ease to help with the current COVID-19 pandemic.

  • With the current pandemic, there has been a limited supply of gear and products that are tailored for fashion and have limited fashion and aesthetic appeal. For example, all the PPP blue masks might not look great on an individual’s outfit, or it looks out of order for the color they are wearing. With our website, we aim to extend the reach of products that are aimed to product individuals from COVID as well as look and feel aesthetically pleasing to the customer.
  • This page is as a collective of products and services that are tailored for COVID, as well as make the consumer want to keep showing off their newest gear to their friends and family

    • Project Plan

    Our project management will follow the agile methodology. Every week we will have a one-hour scrum meeting (every Tuesday at 8 pm) to talk about what we have done the previous week and what we are going to do for the following week. Besides that, we divide the project into modules, thus every week we will have something to deliver. The distribution of the modules throughout the semester will be as follows:

  • Week1 (10/12 – 10/17) - Finish “About Us” tab (proposal and pitch)
  • Week2 (10/19 – 10/24) - Research and design details for layouts and features
  • Week3 (10/26 – 10/31) - Create products listing page:
    • Front-end: Create the UI based on the layouts agreed on
    • Back-end: Program product searching and sorting feature
    • Database: Create template for products data
  • Week4 (11/2 – 11/7) - Create buyers’ account features:
    • Front-end: Create the UI based on the layouts agreed on
    • Back-end: Program functionalities to create an account (sign up), access an account (log in), and modifying shopping lists (adding, deleting, updating products in the cart)
    • Database: Create template for users data
  • Week5 (11/9 – 11/14) - Create transaction page:
    • Front-end: Create the UI based on the layouts agreed on
    • Back-end: Program functionalities to do transactions (payment) and send an email notification to the buyer and seller
    • Database: Create template for users data
  • Week6 (11/16 – 11/21) - Create sellers’ account features:
    • Front-end: Create the UI based on the layouts agreed on
    • Back-end: Program functionalities to create an account (sign up), access an account (log in), and modifying products listing (adding, deleting, updating products the sellers sell).
    • Database: Expand the template for users data
  • Week6 (11/16 – 11/21) - Create sellers’ account features:
    • Front-end: Create the UI based on the layouts agreed on
    • Back-end: Program functionalities to create an account (sign up), access an account (log in), and modifying products listing (adding, deleting, updating products the sellers sell).
    • Database: Expand the template for users data
  • Week7 (11/23 – 11/28) - Polishing and debugging before submission and Finalizing design document

    • Features that may be added if time permits:
  • Adding security plug-ins for users’ accounts and transaction
  • Using cloud services to personalize the web app
      • Different accounts will show different products based on users’ search and transaction history
    Features that may be lost if time limits:
  • Seller accounts (because we are can provide our own products)
